last30days installs on four surfaces: Claude Code, OpenClaw (via clawhub), Gemini CLI, and claude.ai web. All four give you the same research engine and the same zero-config sources out of the box. Pick the tab that matches how you use Claude.

## Requirements

| Requirement | Version | Notes |
|-------------|---------|-------|
| Python | 3.12+ | Required. `brew install python@3.13` on macOS. |
| Node.js | Any LTS | Required for the vendored X search client. |
| yt-dlp | Any | Optional. Unlocks YouTube transcripts. |
| `gh` CLI | Any | Optional. Enables GitHub source and ScrapeCreators registration via GitHub. |

<Warning>
  Gemini CLI v0.9.0 has an upstream installer bug that causes the standard `gemini extensions install` command to fail with `Configuration file not found at /tmp/gemini-extensionXXXXXX/gemini-extension.json`. Use the workaround in the Gemini CLI tab below.
</Warning>

## Installation

<Tabs>
  <Tab title="Claude Code">
    Run the following slash command inside Claude Code:

    ```text
    /plugin marketplace add mvanhorn/last30days-skill
    ```

    The plugin installs immediately. No restart is required. The skill is available as `/last30days` in any Claude Code session.

    To update the skill later:

    ```text
    claude plugin update last30days@last30days-skill
    ```
  </Tab>

  <Tab title="OpenClaw">
    Run the following command in your terminal:

    ```bash
    clawhub install last30days-official
    ```

    clawhub handles the rest. The skill deploys to your OpenClaw environment and is available as `/last30days` immediately after installation completes.
  </Tab>

  <Tab title="Gemini CLI">
    <Warning>
      Gemini CLI v0.9.0 has an upstream installer bug. The standard `gemini extensions install URL` flow fails with a missing configuration file error. Track the upstream issue at [github.com/google-gemini/gemini-cli/issues/11452](https://github.com/google-gemini/gemini-cli/issues/11452). Use the workaround below until a fix ships.
    </Warning>

    Clone the repository locally first, then install from the local path:

    ```bash
    git clone https://github.com/mvanhorn/last30days-skill
    gemini extensions install ./last30days-skill
    ```

    This bypasses the remote installer and reads the extension configuration directly from your local clone.
  </Tab>

  <Tab title="claude.ai web">
    <Steps>
      <Step title="Enable code execution">
        Go to **claude.ai Settings > Capabilities** and enable **Code execution and file creation**. Skills will not run without this setting turned on.
      </Step>
      <Step title="Download the skill file">
        Download `last30days.skill` from the [latest GitHub release](https://github.com/mvanhorn/last30days-skill/releases/latest/download/last30days.skill).
      </Step>
      <Step title="Upload to claude.ai">
        Go to **Settings > Capabilities > Skills** and click the **+** button. Drop the `last30days.skill` file into the upload panel.
      </Step>
      <Step title="Run your first query">
        Open a new Claude conversation and type `/last30days` followed by any topic to start researching.
      </Step>
    </Steps>

    <Note>
      The `.skill` file bundles all scripts into a single portable archive. You don't need Python or Node.js installed locally — claude.ai runs the skill in its own sandboxed environment.
    </Note>
  </Tab>

  <Tab title="Manual / developer">
    Clone the repository directly into your Claude skills directory:

    ```bash
    git clone https://github.com/mvanhorn/last30days-skill.git ~/.claude/skills/last30days
    ```

    The skill is immediately available to any Claude environment that reads from `~/.claude/skills/`.

    To build the `.skill` file for claude.ai web from source:

    ```bash
    bash scripts/build-skill.sh
    ```

    This produces `dist/last30days.skill`, which you can upload via **Settings > Capabilities > Skills** on claude.ai.
  </Tab>
</Tabs>

## Python setup

last30days v3 requires Python 3.12 or later. The skill checks for a compatible interpreter at runtime and exits with a clear error if none is found.

<Tabs>
  <Tab title="macOS">
    ```bash
    brew install python@3.13
    ```

    Homebrew installs to `/opt/homebrew/bin/python3.13` on Apple Silicon or `/usr/local/bin/python3.13` on Intel. The skill auto-detects it.
  </Tab>
  <Tab title="Linux">
    ```bash
    sudo apt install python3.13
    ```

    Or use your distribution's package manager. The skill searches for `python3.14`, `python3.13`, `python3.12`, and `python3` in that order and uses the first one that satisfies the 3.12+ requirement.
  </Tab>
  <Tab title="Windows">
    Download Python 3.13 from [python.org](https://python.org/downloads) and run the installer. Check **Add Python to PATH** during installation.
  </Tab>
</Tabs>

## After installation

Once installed, run `/last30days` to trigger the first-run setup wizard. The wizard takes about 30 seconds and only runs once. It:

1. Detects which tools you already have (yt-dlp, `gh` CLI, browser cookies)
2. Offers to configure X (via browser cookies or an xAI API key) and YouTube (via yt-dlp)
3. Optionally registers a free ScrapeCreators account to unlock TikTok, Instagram, Threads, and Pinterest

Reddit (with comments), Hacker News, Polymarket, and GitHub are always active — no wizard step required.

<Tip>
  You can skip the wizard entirely and start researching immediately. Those four zero-config sources cover a meaningful portion of the available signal for most topics.
</Tip>
